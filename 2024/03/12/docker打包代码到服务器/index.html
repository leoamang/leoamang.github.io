docker打包代码到服务器

发表于2024-03-12|更新于2024-03-27
|阅读量:

Docker使用经验

目标：

​ 把前端代码打包到docker服务器

概要：

​ docker分为客户端（咋们电脑）和服务器端（有ip地址和账号密码的）

​ 客户端： docker build 和docker push

​ 服务器： docker pull和docker run

具体操作：

客户端侧

​ 1、 前端代码侧：

​ 保证成功跑起来没有问题，打包：

​ npm run build

​ 保证打包后的文件夹dist平级下有两个文件：Dockerfile和nginx.conf文件

         - **Dockerfile**

​ FROM nginx:stable-alpine-perl #基镜像

​ RUN apk add --no-cache tzdata && \

​ cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime && \

​ echo "Asia/Shanghai" > /etc/timezone && \

​ apk del tzdata #时区设置，东八区

​ COPY ./dist /app # 把打包好的dist放入app

​ COPY ./nginx.conf /etc/nginx/nginx.conf # 本地配置文件

​ EXPOSE 80 # 暴露端口

​ - nignx.conf

​ user nginx;
​ worker_processes 1;
​ error_log /var/log/nginx/error.log warn;
​ pid /var/run/nginx.pid;
​ events {
use epoll;
​ worker_connections 1024;
​ }
​ http {
include /etc/nginx/mime.types;
default_type application/octet-stream;
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
​ '$status $body_bytes_sent "$http_referer" '
'"$http_user_agent" "$http_x_forwarded_for"';
access_log /var/log/nginx/access.log main;
​ sendfile on;

                        `gzip on;`
                          `gzip_min_length 1k;`
                          `gzip_buffers 4 8k;`
                          `gzip_http_version 1.1;`
                         `gzip_types text/plain application/javascript application/x-javascript text/javascript text/css application/xml;`
  
                        `keepalive_timeout 65;`

​ server {
​ listen 80;

​ #server_name localhost;

                                `access_log /var/log/nginx/access.log main;`

​ location ^~ / {
​ root /app/;
​ index index.html;
​ try_files $uri $uri/ /index.html;
}

                               `#开发模式API`
   
                                `location ^~ /{api/其他}/ {`
      								     `add_header Access-Control-Allow-Origin *;`
          								 `add_header Access-Control-Allow-Methods 'GET, POST, OPTIONS';`
       								    `proxy_pass {后端ip:端口/};`
         								  `proxy_set_header Host $http_host;`
       								    `proxy_set_header X-Real-IP $remote_addr;`
       								    `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;`
       								    `index index.html index.htm index.jsp;`
       							`}`

​ error_page 500 502 503 504 /50x.html;
​ location = /50x.html {
​ root /usr/share/nginx/html;
​ }
}
​ }

2、docker打包并推送镜像：

​ docker build . -t {name}

​ tips:为了加快速度，可以在服务器端设置docker registry设置服务器端内网下的一个地址作 为传输的，{该地址加入daemon.json下的信任路径}这个时候打包代码更新为：

​ docker build . -t {ip}/library/{name}

​ docker push {build . -t之后的内容} # 推送

服务器端：

​ 1、 拉取上传的镜像

​ docker pull {build . -t之后的内容} #拉取客户端上传的代码

​ docker images # 可以查看上传的镜像

​ 2、 创建镜像容器并运行

​ docker run -d -p 8080:80 –-name={自己起} {build . -t之后的内容}

​ docker ps #可以查看容器

​ docker stop {容器名字的前两位} #停止运行容器

​ docker rm {容器名字的前两位} #删除容器

过程中要注意端口的设置，如果这个时候上传了后端，那么前端的nginx.conf代码应该配置后端在服务器暴露的端口（服务器端设置的，例如上run运行的时候的8080）。
